Subagent — order_resolution

Handles: Cases 1011, 1012, 1013, 1014 — every Pronto delivery question.
Looks up the order + courier handoff for the case, decides whether the customer is owed a refund or a goodwill credit per the SLA / wrong-address / missing-items rules, applies it, and replies with a polished HTML card.

Actions

Action 1
get_order_status
Resolves the order tied to the case and returns its current status + courier handoff signals.
Target: apex://GetOrderStatus
Inputs: caseId (string)
Returns:
  • orderNumber — string
  • status — string (e.g. Delivered with issues, Wrong address, Missing items, Not yet delivered)
  • merchantName — string
  • slaMinutesOver — number (0 if on time)
  • recommendedRemedy — string (full_refund | prorated_credit | goodwill_credit | none)
Action 2
apply_refund_or_credit
Issues the refund or credit recommended by Action 1 and confirms the reference id back to the customer.
Target: apex://ApplyRefundOrCredit
Inputs:
  • caseId — string
  • customerEmail — string
  • reason — string (the recommendedRemedy from Action 1)
Returns:
  • referenceId — string (e.g. CR-12345678)
  • creditAmountUsd — number
  • confirmationSentTo — string
